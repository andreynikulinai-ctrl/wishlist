# 🎂 Вишлист — деплой на Vercel

## Что внутри архива

```
wishlist-vercel/
├── api/
│   └── bookings.js          ← бэкенд (serverless)
├── public/
│   ├── index.html           ← сама страница
│   └── images/              ← фото товаров
├── package.json
├── vercel.json
└── README.md (этот файл)
```

## Деплой пошагово

### 1. Создай репозиторий на GitHub

1. Зайди на https://github.com → New repository
2. Назови как хочешь, например `wishlist` или `birthday`
3. Сделай **Private** (чтобы случайно никто не нашёл)
4. Создавай без README

### 2. Залей файлы

Самый простой способ — через сайт GitHub:

1. На странице нового репозитория нажми **"uploading an existing file"**
2. **Распакуй архив** на компьютере
3. Перетащи в окно GitHub **содержимое распакованной папки** (не саму папку!) — то есть файлы `api/`, `public/`, `package.json`, `vercel.json`
4. Внизу нажми **Commit changes**

### 3. Подключи Vercel

1. Зайди на https://vercel.com → **Add New** → **Project**
2. Найди свой репозиторий → **Import**
3. На странице конфигурации **ничего не меняй** → нажми **Deploy**
4. Через 30 секунд получишь ссылку типа `wishlist-abc123.vercel.app`

> **Сайт уже доступен**, но без БД бронирования работать не будут. Идём дальше.

### 4. Подключи базу данных (Upstash Redis)

1. В проекте Vercel вверху → вкладка **Storage**
2. Нажми **Create Database** → выбери **Upstash** → **Redis** → **Continue**
3. Имя: любое (например `wishlist-db`)
4. Регион: **Frankfurt** (ближе всего к РФ)
5. План: **Free** → **Create**
6. Откроется окно подключения → выбери свой проект → **Connect**

Vercel автоматически добавит нужные переменные окружения.

### 5. Передеплой

После подключения БД нужно переразвернуть приложение:
1. В Vercel → вкладка **Deployments**
2. У последнего деплоя нажми три точки `...` → **Redeploy**
3. Подтверди **Redeploy** ещё раз

Через минуту всё заработает.

```
https://wishlist-abc123.vercel.app
```
(точное имя увидишь в Vercel)

### Хочешь красивую ссылку?

В Vercel → Settings → Domains → можешь:
- Поменять название проекта (станет `моё-имя.vercel.app`)
- Или подключить свой домен бесплатно

## Если что-то не работает

**Карточки видны, но при бронировании ошибка:**
- БД не подключена или забыли передеплоить (шаг 5)
- Проверь во вкладке **Logs** что пишет

**Сайт вообще не открывается:**
- Деплой завершился с ошибкой → во вкладке **Deployments** посмотри логи

**Хочу посмотреть кто что забронировал:**
- Vercel → Storage → твоя БД → **Data Browser**
- Найди ключ `wishlist:bookings` — там JSON с ID подарков и временем брони

## Сброс брони

Если нужно очистить (например после ДР повторно тестируешь):
- Vercel → Storage → твоя БД → Data Browser
- Удали ключ `wishlist:bookings` или замени значение на `{}`
